EventForge

Платформа для организации мероприятий

Use Cases
1. Регистрация нового пользователя (Организатор / Участник)
Предусловия

Пользователь впервые открывает приложение.

Триггер

Пользователь нажимает кнопку «Зарегистрироваться».

Основной сценарий

Система предлагает выбрать тип аккаунта:

Организатор

Участник

Система предлагает ввести email (или номер телефона) и пароль.

Пользователь вводит email (или номер телефона) и пароль, соответствующий требованиям безопасности.

Система проверяет, что email (или номер телефона) еще не зарегистрирован.

Система отправляет шестизначный код подтверждения на указанный email (или номер телефона).

Пользователь вводит полученный код в соответствующее поле.

Система подтверждает код и предлагает заполнить профиль (имя, город, интересы).

Пользователь вводит данные профиля.

Система перенаправляет пользователя на главный экран.

Результат

Создан персональный аккаунт пользователя.

Альтернативный сценарий

Если email уже занят, система сообщает: «Аккаунт с таким email уже существует. Попробуйте войти или восстановить пароль».

Если код введен неверно, система отображает ошибку: «Неверный код подтверждения».

2. Создание мероприятия с интеллектуальными подсказками (AI)
Предусловия

Пользователь авторизован как организатор.

Триггер

Пользователь нажимает кнопку «Создать мероприятие» на главном экране.

Основной сценарий

Система открывает форму создания мероприятия с полями:

название,

формат (онлайн / оффлайн / гибрид),

город,

дата и время,

количество гостей,

бюджет.

Пользователь начинает заполнять основные поля.

Интеллектуальный помощник анализирует введенные данные и в реальном времени:

подсказывает оптимальную длительность мероприятия,

рекомендует формат,

предлагает структуру (регистрация, основная программа, перерывы).

Пользователь принимает или отклоняет подсказки.

Пользователь завершает заполнение формы.

Пользователь нажимает кнопку «Опубликовать».

Система сохраняет мероприятие и публикует его в системе.

Результат

В системе создано новое мероприятие с корректно заполненными параметрами.

Альтернативный сценарий

Пользователь игнорирует подсказки и заполняет форму вручную.

Если обязательные поля не заполнены, система выводит сообщение: «Заполните все обязательные поля».

3. Гибкий конструктор программы мероприятия
Предусловия

Пользователь авторизован.
В системе существует созданное мероприятие.

Триггер

Пользователь заходит в карточку мероприятия и нажимает «Программа».

Основной сценарий

Система открывает редактор программы мероприятия.

Пользователь добавляет временные блоки (доклады, шоу, перерывы).

Для каждого блока указывает название, описание, ведущего, время начала и длительность.

Пользователь изменяет порядок блоков с помощью Drag & Drop.

Пользователь нажимает «Сохранить».

Система валидирует временные интервалы и сохраняет программу.

Результат

Создана структурированная программа мероприятия.

Альтернативный сценарий

Если временные блоки пересекаются, система отображает ошибку: «Обнаружено пересечение временных интервалов».

4. Создание и продажа билетов с уровнями доступа
Предусловия

Мероприятие опубликовано.

Триггер

Организатор нажимает кнопку «Настроить билеты».

Основной сценарий

Система открывает форму создания билетов.

Пользователь создает типы билетов (Стандарт, VIP, Онлайн).

Указывает стоимость, количество, период продаж.

Пользователь нажимает «Сохранить».

Участники покупают билеты через приложение.

После покупки билет сохраняется в профиле участника.

Результат

Настроена и запущена продажа билетов.

Альтернативный сценарий

Если лимит билетов исчерпан, система автоматически завершает продажи.

5. Регистрация участников на входе по QR-коду
Предусловия

Пользователь приобрел билет.

Триггер

Пользователь прибывает на мероприятие.

Основной сценарий

Пользователь открывает билет в приложении.

Показывает QR-код сотруднику мероприятия.

Система сканирует код.

Система проверяет валидность билета.

Пользователь отмечается как присутствующий.

Результат

Пользователь успешно зарегистрирован на входе.

Альтернативный сценарий

Если QR-код недействителен, система блокирует вход и уведомляет администратора.

6. Управление бюджетом мероприятия
Предусловия

Пользователь авторизован.
Создано мероприятие.

Триггер

Пользователь открывает раздел «Финансы».

Основной сценарий

Пользователь указывает общий бюджет мероприятия.

Добавляет статьи расходов (площадка, реклама, техника).

Вносит фактические траты.

Система автоматически рассчитывает остаток бюджета.

При превышении лимита система отображает предупреждение.

Результат

Пользователь видит актуальное финансовое состояние мероприятия.

Альтернативный сценарий

Если бюджет не задан, система предлагает создать шаблон бюджета.

7. Командная работа организаторов
Предусловия

Пользователь авторизован.
Создано мероприятие.

Триггер

Пользователь нажимает кнопку «Добавить участника команды».

Основной сценарий

Система предлагает ввести email пользователя.

Пользователь назначает роль (администратор, координатор, модератор).

Пользователь нажимает «Отправить приглашение».

Приглашенный пользователь получает уведомление.

После подтверждения получает доступ к мероприятию.

Результат

К мероприятию подключен новый участник команды.

Альтернативный сценарий

Если приглашение отклонено, система не предоставляет доступ.

8. Умный нетворкинг участников (AI-подбор контактов)
Предусловия

Пользователь зарегистрирован на мероприятие.

Триггер

Пользователь заходит в раздел «Нетворкинг».

Основной сценарий

Пользователь указывает цель участия (бизнес, партнерство, знакомства).

Система анализирует профили участников (сфера деятельности, интересы).

Интеллектуальный модуль формирует список релевантных контактов.

Пользователь выбирает участника и отправляет запрос.

После согласия открывается чат.

Результат

Пользователь получает доступ к релевантным контактам.

Альтернативный сценарий

Если совпадения не найдены, система отображает общий список участников.

9. Массовые уведомления участникам
Предусловия

Существуют зарегистрированные участники.

Триггер

Организатор нажимает кнопку «Отправить уведомление».

Основной сценарий

Организатор вводит текст сообщения.

Выбирает получателей (все / VIP / спикеры).

Нажимает кнопку «Отправить».

Система отправляет push-уведомления.

Результат

Участники оперативно получают сообщение.

Альтернативный сценарий

При отсутствии интернета сообщение доставляется при следующем входе.

10. Формирование итогового отчета по мероприятию
Предусловия

Мероприятие завершено.

Триггер

Организатор нажимает кнопку «Сформировать отчет».

Основной сценарий

Система собирает статистику (посещаемость, билеты, финансы).

Система формирует итоговый отчет в формате PDF.

Пользователь нажимает кнопку «Скачать».

Результат

Сформирован официальный аналитический отчет мероприятия.

Альтернативный сценарий

Если не хватает данных, система сообщает, какие параметры не заполнены.
